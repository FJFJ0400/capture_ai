# RFP - Capture AI Agent 개발 의뢰서

## 프로젝트 범위
- Web App(업로드/인입함/상세/TODO)
- Backend API(Fastify)
- Worker(BullMQ)
- DB(PostgreSQL + Prisma)
- Storage(S3 호환 인터페이스, MVP는 로컬)
이유: MVP를 가장 빠르게 검증하면서도 향후 네이티브 확장 경로를 유지합니다.

## 기능 요구사항
- 멀티 파일 업로드 + 즉시 인입함 반영
- Queue/Worker 기반 비동기 처리(OCR/요약/분류/태그)
- 검색/필터(키워드/카테고리/기간)
- 보안(파일 타입/크기 제한, API Key, 로그 마스킹)
- 삭제권/재처리(FAILED 재시도)
이유: 제품 가치의 핵심 루프와 운영 안정성에 직결되는 기능을 우선합니다.

## 비기능 요구사항
- 성능: 업로드 후 평균 처리 지연 10초 이내(로컬 OCR 기준)
- 안정성: 재시도 3회 및 idempotency 기본 적용
- 관측성: 최소 로그/에러코드 규약으로 원인 추적 가능
- 비용: 로컬 스토리지/오픈소스 OCR 기반
이유: 초기 비용을 낮추고 운영 중 장애를 빠르게 파악할 수 있어야 합니다.

## 산출물
- 모듈 구조/아키텍처 문서
- OpenAPI 명세(요약)
- DB 스키마 문서
- 운영 가이드 및 체크리스트
이유: MVP 이후 확장/이관 시 지식을 남겨 재작업을 줄입니다.
